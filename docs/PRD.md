# PRD：Bamboo Shoot Soup（身份设计师 Agent）

## 1. 产品概述

### 1.1 一句话

一个帮用户“设计第二身份”的 Agent：不是提升你本人，而是构建一个可盈利、可持续、可规模化的虚拟身份系统。

### 1.2 背景与机会

未来 5–10 年，个人会在不同平台拥有多个“人格化身份”来获取注意力与收入（平台算法 + 内容供给过剩 + 身份资产化）。
但多数人缺乏“身份设计能力”：

- 不知道做什么人设、差异化在哪里
- 不知道如何持续输出（风格稳定、叙事主线、成长 arc）
- 不知道如何系统化变现并控制风险（争议、品牌、信任成本）

### 1.3 产品定位

Bamboo Shoot Soup 是一个**专门帮助用户构建“算法适配人格（可规模化人格）”**的 Agent，输出从“身份模型 → 内容系统 → 增长实验 → 变现路径 → 风险控制”的全链路方案，并在后续持续辅助执行。

## 2. 目标与非目标

### 2.1 产品目标（Goals）

- 让用户在 30–60 分钟内得到 3–5 个可落地身份模型（可选择其一启动）
- 让用户在 7 天内完成身份启动包（账号定位 + 首批内容结构 + 发布节奏）
- 让用户在 4 周内形成人格一致性与内容流水线（持续输出不崩）
- 让用户在 8–12 周内产出至少 1 条可验证变现路径（订阅/咨询/课程/工具/品牌合作等）

### 2.2 非目标（Non-goals）

- 不做通用“自我提升/心理疗愈”类陪伴
- 不承诺快速爆红；更关注长期资产化与可控增长
- 不替用户代运营账号（可提供模板/脚本/实验设计与决策建议）

## 3. 目标用户与场景

### 3.1 目标用户（Personas）

- 职场创作者：有专业能力但表达体系不稳定，想用内容变现
- 转型者/副业探索者：想构建可盈利第二曲线，缺少定位方法
- 匿名输出者：想做行业分析号、商业观察号，担心身份风险
- 多平台经营者：在小红书/Twitter/视频号/公众号等多平台割裂输出，缺少统一叙事
- 高客单服务者（咨询/教练/顾问/设计师/自由职业者）：想把人设变成可复制的获客系统

### 3.2 核心使用场景

- “我该用什么身份做内容？怎么不撞人设？”
- “我坚持不下去：今天写什么？口吻怎么保持一致？”
- “这条内容要不要发？会不会翻车？对长期信任有啥影响？”
- “我怎么把这个身份变成收入？走哪条路径最快验证？”

## 4. 用户旅程与关键体验

### 4.1 用户旅程（Journey）

1. Onboarding 诊断：输入/问答 → 能力结构与约束条件建模
2. 身份生成：产出 3–5 个身份候选 → 用户选择 1 个主身份 + 1 个备选
3. 身份启动包：定位陈述、价值主张、内容支柱、语气词典、禁区清单、增长假设
4. 内容系统：每周主题 + 内容矩阵 + 脚本草稿 + 系列化规划
5. 一致性引擎：持续校准风格、观点主线、成长 arc，避免人格漂移
6. 增长实验：设计 1–3 个实验（标题/选题/发布频率/互动策略），迭代
7. 变现路径：根据数据与用户反馈进入变现试验（订阅/产品化/咨询/合作）
8. 人格模拟器（进阶）：发文前风险/争议/增长预测，给出决策建议

### 4.2 北极星体验（North Star Experience）

用户能清晰说出：

“我是谁（这个身份）—我输出什么—我怎么持续—我怎么赚钱—我怎么避免翻车。”

## 5. 产品功能设计

下面按 MVP（最小可用）→ V1 → V2 分阶段。

### 5.1 MVP 功能（可 2–4 周内上线）

目标：先让用户拿到可落地身份模型 + 7 天启动包。

#### 身份诊断（能力结构分析）

- 输入方式：问卷 + 对话式追问（10–20 分钟）
- 关键维度：
  - 技能栈（硬技能/软技能/资源）
  - 兴趣与能量曲线（能坚持的内容类型）
  - 认知风格（拆解型/叙事型/观点型/工具型）
  - 价值观与边界（哪些话题不能碰）
  - 风险承受（争议、曝光、实名/匿名）
  - 时间投入（每周可投入小时数）
- 输出：能力画像卡（优势/弱项/适合平台/适合内容形态）

#### 3–5 个身份模型生成器

每个身份模型必须包含（结构固定，方便比较）：

- 身份标题（可记忆）
- 受众是谁 + 核心痛点
- 内容方向（3–5 个内容支柱）
- 语气风格（关键词 + 示例句）
- 核心观点库（5–10 条“长期立场”）
- 差异化定位（与常见同类账号差在哪）
- 成长路径（0–3 月/3–12 月）
- 变现路径图（从 0 到 1 的验证顺序）
- 风险与禁区（可能翻车点、法律/平台风险提醒）

#### 人格一致性引擎（基础版）

产出“人格宪法”：

- 口吻词典：常用词/禁用词/句式偏好
- 观点护城河：3 条不可动摇的世界观
- 叙事主线：我为什么做这件事（持续动机）
- 成长 arc：从新手到权威的阶段叙事模板

提供“一致性检查器”（输入草稿 → 输出偏离点提示）。

#### 7 天启动包（自动内容系统·轻量版）

- 每日主题（7 个）
- 每日 1 篇内容草稿（或大纲 + 开头 300 字）
- 3 个系列栏目建议（可持续）
- 1 个增长实验建议（如标题 A/B、互动提问、评论区话术）

MVP 交付物：用户选择的主身份 + 人格宪法 + 7 天内容包。

### 5.2 V1 功能（可 6–10 周迭代）

目标：把“持续输出”变成系统，把“变现”变成路径。

#### 内容矩阵与系列化工厂

- 设定 3–5 个内容支柱 → 自动生成：
  - 系列选题池（每个支柱 20–50 个）
  - 栏目结构（例如：每周一趋势拆解、周三工具、周五复盘）
  - 多平台适配（同主题改写为小红书/推特线程/公众号）
- 支持用户反馈：哪些题“写得动/数据好” → 强化该支柱权重

#### 增长实验面板（轻量数据闭环）

- 用户手动录入或平台数据导入（后期）
- 指标跟踪：发布频率、互动率、收藏/转发、私信线索
- 实验管理：假设 → 方案 → 结果 → 结论 → 下次迭代

#### 变现路径规划器

根据身份类型推荐变现策略：

- 订阅/会员
- 咨询/服务产品化
- 数字产品（模板/课程/工具）
- 品牌合作
- 引流到站外（邮件列表/社群/网站）

提供“12 周变现路线图”：每周要做什么、产出什么、验证什么。

#### 身份组合策略（3–5 身份系统）

- 主身份（稳定输出）
- 备身份（实验）
- 匿名身份（敏感洞察）
- 工具身份（引流/产品销售）

规则：如何避免互相冲突、如何复用内容资产。

### 5.3 V2 功能（进阶 Agent：人格模拟器）

目标：在关键决策点提供“预测 + 风险控制 + 长期信任”建议。

#### 人格模拟器（发布前评估）

输入：草稿 + 平台 + 当前阶段目标  
输出：

- 粉丝增长曲线预测（区间而非确定值）
- 评论争议概率（低/中/高 + 触发点）
- 品牌风险提示（可能踩的雷）
- 长期信任影响（是否与人格宪法冲突）
- 建议：发/不发/怎么改（给出改写版本）

#### “观点资产库”自动生长

- 从用户输出中提炼观点、案例、方法论
- 自动形成可复用组件：金句、框架、案例库、FAQ

## 6. 关键交互与输出模板

### 6.1 核心页面/模块（如果做成产品形态）

- 诊断中心：问卷 + 画像卡
- 身份工坊：身份候选对比（卡片式）
- 人格宪法：可编辑的规则与词典
- 内容工厂：选题池/周计划/草稿生成
- 实验面板：实验列表 + 指标记录
- 变现路线图：12 周计划 + 任务清单
- 发布前检查（V2）：人格模拟器输出

### 6.2 标准化交付物（强制结构）

- Identity Model Card（身份模型卡）
- Persona Constitution（人格宪法）
- 7-Day Launch Kit（7 天启动包）
- Content Pillars & Matrix（内容支柱与矩阵）
- Monetization Map（变现路径图）
- Risk & Boundary List（风险与边界清单）

## 7. 数据指标与成功标准

### 7.1 北极星指标（NSM）

用户在 7 天内完成“身份启动包并发布 ≥2 条内容”的比例。

### 7.2 核心漏斗指标

- Onboarding 完成率
- 身份模型选择率（从 3–5 选 1）
- 7 天内容包使用率（生成后被实际发布/使用的比例）
- 一致性检查使用率（每周触发次数）
- 4 周留存（W4 retention）
- 变现验证率（8–12 周内出现第一笔收入或明确线索）

### 7.3 质量指标（需要防“爽文输出”）

- 用户对“可执行性”的评分（是否能照着做）
- 身份差异化评分（“像不像别人”反向指标）
- 输出一致性评分（随时间漂移程度）

## 8. 风险与对策

### 同质化输出风险（AI 常见问题）

对策：差异化强制字段（观点库/禁区/个人经历素材）、反模板机制（生成时要求“反主流角度/小众受众/独特叙事”至少一项）。

### 用户输入太少导致模型空泛

对策：对话式追问；提供“素材采集任务”（列 10 个经历/3 个失败/5 个观点）。

### 平台/法律/伦理风险（匿名、敏感话题、虚假身份）

对策：风险承受问卷 + 边界清单 + 发布前风险提示；明确“禁止违法/冒充/侵权”的规则提醒。

### 执行疲劳：用户拿到方案但不做

对策：7 天极简启动、周计划、任务颗粒度小、反馈闭环（记录发布→系统调整）。

## 9. 商业模式与定价框架（建议）

### 创作者订阅版（核心）

月/年订阅：身份迭代 + 内容系统 + 一致性检查 + 实验建议。

### 高端 IP 打造版（高客单）

4–8 周项目制：深度访谈 + 身份定制 + 变现产品设计 + SOP。

### 企业高管形象版

职业风险更高：风格更稳健 + 合规边界 + PR 风险提示。

### 匿名创业者版 / 出海版

匿名策略、跨文化表达、平台适配（Twitter/LinkedIn 等）。

## 10. MVP 里程碑（建议拆解）

- M0：身份诊断问卷 + 画像卡输出
- M1：3–5 身份模型生成（结构化模板）
- M2：人格宪法生成 + 一致性检查（文本级）
- M3：7 天游程内容包（主题 + 大纲/草稿）
- M4：内测指标验证（7 日发布率、W4 留存、可执行性评分）

## 11. 附录：MVP 需要的关键 Prompt / 规则（示例方向）

身份生成必须遵守：

- “差异化”字段不可为空（必须写清与同类区别）
- “变现路径”必须给出验证顺序（先做什么、怎么验证）
- “语气风格”必须包含 5 句示例口吻
- “风险与禁区”必须结合用户的风险承受度输出
