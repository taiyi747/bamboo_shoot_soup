# Hackathon Demo Script (5 min)

## 1) 演示前准备（2-3 分钟）

1. 启动后端：`uvicorn app.main:app --reload`
2. 启动前端：`cd frontend && bun run dev`
3. 打开 `http://127.0.0.1:3000`
4. 确认 `.env`：
   - `DEMO_REPLAY_FALLBACK_ENABLED=true`
   - `DEMO_REPLAY_FORCE=false`

## 2) 5 分钟口播节奏

### 第 0-1 分钟：问题与定位

- 这是一个“身份设计师 Agent”，面向职场创作者。
- 目标不是给模板，而是给“身份-内容-实验-变现”的执行闭环。

### 第 1-2 分钟：MVP 生成链路

- 走 Onboarding -> 身份模型 -> 人格宪法 -> 启动包 -> 一致性检查。
- 强调规则化输出：差异化、语气示例、风险边界都有硬校验。

### 第 2-4 分钟：V1 展示闭环

- 进入 `内容矩阵` 页面，展示支柱选题和平台改写。
- 进入 `实验面板` 页面，创建实验并填写结果/结论。
- 进入 `变现路线图` 页面，展示 12 周任务与验证指标。

### 第 4-5 分钟：交付与可追踪性

- 进入 `交付汇总` 页面，展示全链路产物与事件轨迹。
- 导出 JSON 交付包，说明可复盘、可审计、可继续迭代。

## 3) 故障切换脚本（断网/限流）

1. 将 `.env` 改为：`DEMO_REPLAY_FORCE=true`
2. 重启后端
3. 重新走关键生成按钮
4. 说明：当前结果来自最近一次成功回放，确保现场稳定展示

## 4) 演示后验证

- 后端：`pytest`
- 前端：`cd frontend && bun run test`

## 5) 常见答辩问题答法

- 为什么是本地优先：隐私、低延迟、可离线演示。
- 为什么加回放：黑客松现场稳定性优先，避免网络抖动导致断演示。
- 为什么强调实验结论：防止“无结论迭代”，保证业务闭环可执行。
